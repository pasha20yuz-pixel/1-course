# Лабораторная работа №4: Разбор и вычисление математических выражений с построением AST

**Дисциплина:** Структуры данных  
**Семестр:** 2  

## Цель работы
Закрепление навыков работы с динамической памятью в C, реализация абстрактного синтаксического дерева (AST) для представления выражений, разработка алгоритмов разбора инфиксных выражений (сортировочная станция), преобразование в префиксную/постфиксную формы, вычисление с подстановкой переменных, контроль утечек памяти.

## Требования варианта (№5)
- **Числа:** целые 32-битные (0 … 2147483647).
- **Переменные:** латинские буквы/цифры.
- **Бинарные операции:** `+`, `-`, `*`, `/`, `%` (целочисленные), `^` (возведение в степень, правый аргумент ≥ 0), `#` (НОД).
- **Унарные операции (дополнительно):** унарный минус `-` и унарный плюс `+` (приоритет выше всех бинарных, могут стоять перед числом, переменной или подвыражением).
- **Приоритеты (от высшего к низшему):** унарные (`-` `+`) > `#` > `^` > `* / %` > `+ -`.
- **Ассоциативность:** все бинарные – левоассоциативны, кроме `^` (правоассоциативен). Унарные – правоассоциативны (традиционно).

## Команды программы
- `parse <выражение>` — разобрать инфиксное выражение, построить AST, вывести `success` или `incorrect`.
- `load_prf <префиксная_запись>` — загрузить дерево из префиксной формы.
- `load_pst <постфиксная_запись>` — загрузить дерево из постфиксной формы.
- `save_prf` — вывести текущее дерево в префиксной форме (или `not_loaded`).
- `save_pst` — вывести в постфиксной форме.
- `eval [переменная=значение ...]` — вычислить выражение с подстановкой.
- `exit` — завершить работу.

## Содержание папки
- `ast.c` – определение узлов AST, создание/удаление, обходы в префикс/постфикс, вычисление (поддерживает унарные операции).
- `parser.c` – лексер, алгоритм сортировочной станции с поддержкой унарных операций, префиксный и постфиксный парсеры.
- `main.c` – обработка команд, чтение из `input.txt` или stdin, вывод `memstat.txt`.
- `memstat.h` – макросы для подсчёта malloc/free.
- `input.txt` – пример команд.
- `Лабораторная_СД_4.docx` – отчёт с теорией, тестами, выводами.

Поддержка унарного минуса и плюса (реализация)
В текущей версии кода добавлена обработка унарных операций. Основные изменения:

1. Лексер (parser.c)
Функция lexer_next() теперь распознаёт унарный контекст. В алгоритме сортировочной станции при обнаружении - или + проверяется, является ли оператор унарным (предыдущий токен – оператор, (, начало выражения). Унарный оператор помещается в стек операторов со специальным маркером UMINUS/UPLUS.

2. AST (ast.c)
Добавлен новый тип узла NODE_UNARY_OP:

c
typedef enum {
    NODE_NUMBER,
    NODE_VARIABLE,
    NODE_BINARY_OP,
    NODE_UNARY_OP        // для унарного минуса/плюса
} NodeType;

typedef struct AstNode {
    NodeType type;
    union {
        long long value;
        char var_name[64];
        struct {
            char op;               // для бинарной операции
            struct AstNode* left;
            struct AstNode* right;
        } binary;
        struct {
            char op;               // '+' или '-'
            struct AstNode* operand;
        } unary;
    };
} AstNode;
3. Алгоритм сортировочной станции
При выталкивании унарного оператора создаётся узел create_unary_node(op, operand). В обходах to_prefix/to_postfix унарный узел выводится как "operator operand" (например, - 5).

4. Вычисление
c
case NODE_UNARY_OP:
    if (node->unary.op == '-') return - evaluate(node->unary.operand, ...);
    else return evaluate(node->unary.operand, ...); // унарный плюс ничего не меняет
5. Пример работы
Вход: parse -5+-(2-3)

Дерево: бинарный +, левый операнд – унарный минус от 5, правый – унарный минус от (2-3).

Префиксная форма: + - 5 - - 2 3

Вычисление при eval: (-5) + (-(2-3)) = -5 - (-1) = -4

Примечания
Программа подсчитывает вызовы malloc/free через макросы и пишет memstat.txt при выходе.

При синтаксической ошибке выводится incorrect, при отсутствии дерева – not_loaded.

Унарный минус имеет самый высокий приоритет (10), унарный плюс (9) – в текущей реализации они оба имеют приоритет 9, что выше бинарных.

Операция ^ правоассоциативна: 2^3^2 = 2^(3^2) = 512.

Поддерживаются переменные (латинские буквы/цифры).

Тип данных для вычислений – long long (64 бита), но входные числа ограничены 32 битами по условию.

Лицензия
Учебный код, свободное использование в образовательных целях.
