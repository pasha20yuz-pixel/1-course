# Лабораторная работа №3: База данных метеорологических станций (односвязный список)

**Дисциплина:** Структуры данных  
**Семестр:** 2  
**Вариант:** 5 – сеть метеорологических станций

## Цель работы
Закрепление навыков работы с динамической памятью, реализация односвязного списка для хранения записей, разработка команд вставки, выборки, удаления, обновления, удаления дубликатов, а также экспорта/импорта в CSV.

## Команды программы
- `insert field1=value1,field2=value2,...` – вставка новой записи (все 7 полей обязательны).
- `select field1,field2,... [условия]` – вывод запрошенных полей записей, удовлетворяющих условиям (AND).
- `delete [условия]` – удаление записей по условиям.
- `update field1=value1,field2=value2,... [условия]` – обновление указанных полей для записей, удовлетворяющих условиям.
- `uniq field1,field2,...` – удаление дубликатов (оставляется последняя запись в группе).
- `export filename.csv` – экспорт всех записей в CSV-файл.
- `import filename.csv` – импорт записей из CSV-файла (замена текущей БД).
- `exit` – завершение работы.

## Поля записи

| Поле      | Тип                     | Описание                        |
|-----------|-------------------------|---------------------------------|
| geo_id    | int                     | идентификатор станции           |
| geo_pos   | string (до 255 символов)| местоположение                  |
| mea_date  | date (dd.mm.yyyy)       | дата наблюдений                 |
| level     | int                     | высота над уровнем моря (м)     |
| sunrise   | time (hh:mm)            | время восхода                   |
| weather   | enum (fair,rain,cloudy,snow) | погода                       |
| sundown   | time (hh:mm)            | время захода                    |

## Условия
Поддерживаются операторы: `=`, `!=`, `<`, `<=`, `>`, `>=`, а также специальный оператор `/in/` для проверки вхождения в список: `weather/in/['fair','rain']`.

## Сборка и запуск

### Требования
- Компилятор C99 (gcc, clang, MSVC)
- ОС: Windows / Linux / macOS

### Компиляция
```bash
cd lab3
gcc -std=c99 main.c src/list.c src/cmd.c src/memstat.c -Iinclude -o lab3.exe
```
### Запуск
Пакетный режим: lab3.exe input.txt – команды читаются из файла.

Интерактивный режим: lab3.exe – команды вводятся с клавиатуры (приглашение :>).

### Пример использования (файл input.txt)
```text
insert geo_id=1,geo_pos=Moscow,mea_date=15.03.2024,level=120,sunrise=06:30,weather=fair,sundown=18:45
insert geo_id=2,geo_pos=St.Petersburg,mea_date=15.03.2024,level=10,sunrise=07:00,weather=rain,sundown=19:00
select geo_id,geo_pos,weather
export stations.csv
import stations.csv
select geo_id,geo_pos,weather
exit
```
### Дополнительное задание (экспорт/импорт CSV)
Экспорт: все записи сохраняются в CSV-файл с заголовками. Поля с запятыми или кавычками экранируются.

Импорт: заменяет текущую БД данными из CSV, проверяет корректность типов и наличие всех полей. Чтение построчное, подходит для больших объёмов.

### Примечания
Программа подсчитывает вызовы malloc/free и пишет статистику в memstat.txt.

При синтаксической ошибке выводится incorrect и первые 20 символов команды.

Утечек памяти нет (баланс malloc/free).
