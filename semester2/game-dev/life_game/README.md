# Conway's Game of Life with Barriers and Campaign

## Описание проекта

Это графическая реализация классической игры «Жизнь» Конвея с дополнительными механиками:

- **Классический режим** – поле 25×25, управление скоростью, пауза, пошаговый режим, загрузка паттернов (планер, мигалка, случайное заполнение). Поддерживается **рисование клеток мышью**: левая кнопка – поставить живую клетку, правая – удалить.
- **Кампания** – 10 уровней возрастающей сложности. На каждом уровне игрок должен разместить заданное количество стартовых клеток (не более лимита) так, чтобы все клетки поля заразились за указанное число поколений. Есть кнопка **Auto Solve** для случайной расстановки.
- **Барьеры** – непроницаемые перегородки между соседними клетками (горизонтальные и вертикальные). Механизм реализован, но в текущих уровнях барьеры не используются (будущее расширение).
- **UNDO** – в кампании можно отменить последнее действие (размещение клетки или шаг симуляции) клавишей **U**. Откат реализован через **связный список** состояний (динамическая структура данных).
- **Сохранение прогресса** – последний открытый уровень сохраняется в `progress.dat`.
- **Логирование** – все ключевые события записываются в `game.log` с временной меткой.
- **Справка и «О программе»** – доступны из главного меню.

## Технологии

- Язык: C (стандарт C99)
- Графическая библиотека: [raylib](https://www.raylib.com/) (v5.0+)
- Компилятор: GCC (MinGW-w64 для Windows)
- Система сборки: `gcc` из командной строки или задачи VS Code (`tasks.json`)

## Используемые структуры данных

- `enum` – для режимов игры, фаз кампании.
- `struct` – для группировки данных (уровень, барьеры, состояние игры, кнопки).
- `union` – в структуре уровня (демонстрация, поле `extra`).
- **Связный список** – для хранения истории UNDO (узлы `HistoryNode`).

## Установка и запуск (Windows)

### 1. Установите raylib

- Скачайте сборку для MinGW с [официального GitHub](https://github.com/raysan5/raylib/releases) (например, `raylib-5.5_win64_mingw-w64.zip`).
- Распакуйте в `C:\raylib`. Внутри должны быть папки `include` и `lib`.

### 2. Установите компилятор MinGW-w64

- Например, через MSYS2 или отдельный дистрибутив (mingw-w64). Убедитесь, что `gcc` доступен в `PATH`.

### 3. Склонируйте репозиторий

```bash
git clone https://github.com/pasha20yuz-pixel/1-course.git
cd 1-course/semester2/game-dev/life_game
```
### 4. Скомпилируйте игру
```bash
gcc -std=c99 -D_WIN32_WINNT=0x0601 -I C:/raylib/include -L C:/raylib/lib src/main_raylib.c src/patterns.c src/file_io.c src/world.c src/barriers.c src/levels.c -lraylib -lopengl32 -lgdi32 -lwinmm -lm -o game.exe
```
Если вы используете Visual Studio Code, просто нажмите Ctrl+Shift+B (задача build настроена в .vscode/tasks.json).

### 5. Запустите
```bash
.\game.exe
```

##### Классический режим

| Клавиша | Действие |
|---------|----------|
| Пробел | Пауза / продолжение |
| Enter | Один шаг (на паузе) |
| + / - | Увеличить/уменьшить скорость |
| G | Показать/скрыть сетку |
| F1 | Случайное заполнение (плотность зависит от выбранной сложности) |
| F2 | Планер (Glider) |
| F3 | Мигалка (Blinker) |
| Левая кнопка мыши | Поставить живую клетку |
| Правая кнопка мыши | Удалить клетку |

##### Кампания (фаза размещения)

| Действие | Результат |
|----------|-----------|
| Левая кнопка мыши | Поставить стартовую клетку (если не превышен лимит) |
| Правая кнопка мыши | Удалить стартовую клетку |
| Кнопка **Start** | Начать симуляцию |
| Кнопка **Reset** | Очистить поле и сбросить поколение |
| Кнопка **Auto Solve** | Случайная расстановка стартовых клеток |
| Клавиша **U** | Отменить последнее действие (Undo) |
| Кнопка **Back to Menu** | Вернуться в главное меню |

#### Уровни

Все уровни сбалансированы так, чтобы при разумном размещении стартовых клеток поле заражалось за указанное число поколений. Параметры:

| Уровень | Размер | Стартовых клеток | Целевых поколений |
|---------|--------|------------------|--------------------|
| 1 | 4×4 | 3 | 4 |
| 2 | 6×6 | 3 | 6 |
| 3 | 8×8 | 4 | 8 |
| 4 | 10×10 | 5 | 10 |
| 5 | 12×12 | 6 | 12 |
| 6 | 14×14 | 7 | 14 |
| 7 | 16×16 | 8 | 16 |
| 8 | 18×18 | 9 | 18 |
| 9 | 20×20 | 10 | 20 |
| 10 | 22×22 | 11 | 22 |

> **Примечание:** барьеры в текущих уровнях отсутствуют (все матрицы заполнены нулями), но механизм полностью работоспособен и может быть использован для создания более сложных карт.

# Возможные проблемы

- **Не компилируется:** проверьте пути к raylib (`-I` и `-L`). Убедитесь, что используется правильный флаг `-D_WIN32_WINNT=0x0601`.
- **Не работают цвета в консоли:** запускайте игру из терминала, поддерживающего ANSI-коды (например, Windows Terminal).
- **Не загружаются уровни:** папка `levels` должна находиться в той же директории, что и `game.exe`. Убедитесь, что файлы `level1.txt` … `level10.txt` присутствуют.
- **Undo не работает:** в кампании используйте клавишу **U** (не Ctrl+Z). Проверьте, что файл `game.log` создаётся – это косвенно подтвердит, что программа запускается.
